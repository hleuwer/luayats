Protocol Simulation

Spanning Tree Protocol

Luayats uses a modified version of rstplib written by Alex Rozin and Michael Rozhavsky. This library implements an IEEE802.1D-2004 compliant Rapid Spanning Tree algorithm. The protocol state machines are wrapped into a Luayats compliant network object called yats.bridge, that can be instantiated following normal Luayats rules. The bridge network object does only handle protocol data units (PDUs) and does not implement a full featured 802.1D transparent bridge.

An arbitrary number of bridges can be instantiated to simulate large networks. Configuration of the spanning tree and the bridge ports occurs via methods of the bridge class.

A simple example can be found here. See also the examples for more sophisticated network examples.

A bridge can also be configured to run per VLAN Spanning Tree Protocol (PVST) for an arbitrary number of VLANs.

The bridge provides an extensive set of service primitives to trace BPDU flow and state changes of the RSTP protocol state machine.
